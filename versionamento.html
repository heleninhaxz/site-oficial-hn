Vercionamento Git Bash

Conceito: O versionamento com Git Bash é o processo de controlar e gerenciar mudanças em arquivos e projetos usando o Git via linha de comando.

Vercionamento no Git Bash - Funções Básicas

  • Inicialização de repositório: Use git init para começar a rastrear um diretório.

  • Adicionar arquivos: git add prepara arquivos para serem incluídos no próximo commit.

  • Commit de mudanças: git commit -m "mensagem" salva alterações no repositório.

  • Verificar status: git status mostra o estado atual dos arquivos no repositório.

  • Branching e merging: Crie, alterne e funda branches usando git branch, git checkout e git merge.

  • Sincronização remota: git remote, git push e git pull para interagir com repositórios remotos.

Vercionamentos - Curiosidades

  • Snapshot-based: O Git é um sistema de controle de versão distribuído e snapshot-based. Isso significa que ele não armazena alterações de arquivos, mas sim snapshots (instantâneos) do estado do projeto em diferentes pontos no tempo. Isso permite uma rápida criação e aplicação de branches, merges e outras operações, além de garantir a integridade dos dados.

  • Branches Leves: Uma das características marcantes do Git é a facilidade e leveza com que se pode criar e gerenciar branches. Isso incentiva o desenvolvimento paralelo de recursos e experimentação sem comprometer a estabilidade do código principal. Além disso, a fusão de branches no Git é geralmente rápida e eficiente.

  • Repositórios Remotos: O Git facilita o trabalho colaborativo ao permitir a criação de repositórios remotos, como no GitHub ou GitLab, onde equipes podem compartilhar e colaborar em projetos de software. O Git Bash permite interagir com esses repositórios remotos de forma transparente, facilitando o desenvolvimento colaborativo.

  • Revertendo Commits: O Git oferece várias maneiras de reverter commits, seja desfazendo alterações locais não confirmadas, revertendo commits específicos ou redefinindo o estado do repositório para um commit anterior. Essa flexibilidade permite corrigir erros rapidamente e manter um histórico de alterações limpo e organizado.
